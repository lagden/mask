# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

`@tadashi/mask` — a tiny, dependency-free ESM library for input masking in the browser. The entire library is a single file: `src/mask.js`. Tests run in Node
using jsdom to simulate the DOM.

## Commands

Requires both Node (npm scripts, tests) and Deno (lint/format).

- `npm test` — full pipeline: `pretest` (deno fmt --check, deno lint, tsc type-check) → c8 coverage over `node --test test/*.spec.js` → `posttest` (rollup CJS
  build)
- `node --test test/mask.spec.js` — run tests only, without lint/build
- `node --test --test-name-pattern="input init" test/mask.spec.js` — run a single test
- `npm run lint` / `npm run check` / `npm run format` — deno lint / fmt check / fmt write (config in `deno.jsonc`: tabs, single quotes, no semicolons, line
  width 160)
- `npm run tsc` — type-checks the JSDoc-annotated JS and emits `types/mask.d.ts` (declaration-only, `checkJs: true`)
- `npm run rollup` — builds `dist/mask.cjs` (CommonJS) from `src/mask.js`

## Architecture

`src/mask.js` exports a single default `Mask` class:

- `Mask.core(value, mask)` — pure static masking function. Mask tokens: `9` (digit), `A` (alphanumeric), `S` (letter); any other character is a literal.
  `Mask.masking()` is a deprecated alias kept for backward compatibility.
- `new Mask(input, opts)` — binds to an `HTMLInputElement` and masks on events (`keyEvent`, default `input`; optionally `blur`). Instances are tracked in a
  module-level `WeakMap` keyed by the element; constructing twice on the same input throws — use `Mask.data(input)` to retrieve an existing instance, and
  `destroy()` to unbind (which also unmasks the value).
- The mask source can be a string, the element's `data-mask` attribute, a function `(input) => mask`, or a two-element array (switches to the second mask when
  the value outgrows the first, e.g. CPF/CNPJ). `dynamicDataMask: true` watches `data-mask` via `MutationObserver` and re-masks on change.
- Browser globals are accessed via `globalThis` (`HTMLInputElement`, `MutationObserver`), which is what lets tests inject jsdom equivalents onto `globalThis`.

Types are not hand-written: `types/mask.d.ts` is generated by tsc from JSDoc. Edit the JSDoc in `src/mask.js`, not the `.d.ts`.

## Conventions

- Formatting and linting come from Deno (`deno.jsonc`), not ESLint/XO/Prettier. Run `npm run format` before committing; CI (`npm test`) fails on format drift.
- `dist/` and `types/` are build artifacts regenerated by `posttest`; don't edit them directly.
