# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

`@tadashi/mask` ‚Äî a tiny, dependency-free ESM library for input masking in the browser. The entire library is a single file: `src/mask.js`. Tests run via Deno
using jsdom to simulate the DOM.

## Commands

Requires Deno. All tasks are defined in `deno.jsonc`.

- `deno task test` ‚Äî full pipeline: fmt check ‚Üí lint ‚Üí tsc type-check ‚Üí deno test with coverage ‚Üí rollup CJS build
- `deno test -A test/mask.spec.js` ‚Äî run tests only, without lint/build
- `deno test -A --filter="input init" test/mask.spec.js` ‚Äî run a single test
- `deno task lint` / `deno task check` / `deno task format` ‚Äî deno lint / fmt check / fmt write (config in `deno.jsonc`: tabs, single quotes, no semicolons,
  line width 160)
- `deno task tsc` ‚Äî type-checks the JSDoc-annotated JS and emits `types/mask.d.ts` (declaration-only, `checkJs: true`)
- `deno task rollup` ‚Äî builds `dist/mask.cjs` (CommonJS) from `src/mask.js`

## Architecture

`src/mask.js` exports a single default `Mask` class:

- `Mask.core(value, mask)` ‚Äî pure static masking function. Mask tokens: `9` (digit), `A` (alphanumeric), `S` (letter); any other character is a literal.
  `Mask.masking()` is a deprecated alias kept for backward compatibility.
- `new Mask(input, opts)` ‚Äî binds to an `HTMLInputElement` and masks on events (`keyEvent`, default `input`; optionally `blur`). Instances are tracked in a
  module-level `WeakMap` keyed by the element; constructing twice on the same input throws ‚Äî use `Mask.data(input)` to retrieve an existing instance, and
  `destroy()` to unbind (which also unmasks the value).
- The mask source can be a string, the element's `data-mask` attribute, a function `(input) => mask`, or a two-element array (switches to the second mask when
  the value outgrows the first, e.g. CPF/CNPJ). `dynamicDataMask: true` watches `data-mask` via `MutationObserver` and re-masks on change.
- Browser globals are accessed via `globalThis` (`HTMLInputElement`, `MutationObserver`), which is what lets tests inject jsdom equivalents onto `globalThis`.

Types are not hand-written: `types/mask.d.ts` is generated by tsc from JSDoc. Edit the JSDoc in `src/mask.js`, not the `.d.ts`.

## Conventions

- Formatting and linting come from Deno (`deno.jsonc`), not ESLint/XO/Prettier. Run `deno task format` before committing; CI (`deno task test`) fails on format
  drift.
- `dist/` and `types/` are build artifacts regenerated by `deno task test`; don't edit them directly.
